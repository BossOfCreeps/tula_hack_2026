# Задание

TEAM BUILDER  
(сервис для руководителя по формированию профиля сотрудника)

## Проблематика:

Руководитель не всегда понимает какой сотрудник ему нужен в команду для эффективной и слаженной работы, а также определения условий и инструментов мотивации по работе с действующим составом

## Задача:

Разработать сервис-помощник (конструктор) для руководителя при помощи которого руководитель сможет формировать профиль для подбора и расстановки сотрудников в своей команде, а также получать готовые рекомендации по работе с ними и созданию необходимых условий

## Механика:

Руководитель имеет возможность на основании критериев:
- роль в команде
- типология личности (DISC)
- мотивационный профиль
- возраст/поколение

Получить рекомендованный профиль для подбора сотрудника либо по работе с действующим составом при известных вводных (текущий состав и свой профиль как руководителя)

Выбор инструментов и способ реализации оставляем за рабочей группой.

# Как запустить

* Установить Python 3.13
* Установить PostgreSQL
* Скачать код
* Загрузить зависимости из requirements.txt
* Сделать копию файла .env.example -> .env
* Заполните переменные окружения (ключи GIGACHAT можно получить https://developers.sber.ru/)
* Сделать миграцию `python manager.py migrate`
* Запустить `python manager.py runserver`
* Создать пользователя `python manage createsuperuser`
* Авторизоваться в админке http://localhost:8000/admin/
* Создать пользователей http://localhost:8000/admin/users/role/
* Создать роли http://localhost:8000/admin/users/role/
* Открыть корень сайта http://localhost:8000

# Реализованный функционал

В рамках проекта создал систему, которая анализирует команду, её требуемые и текущие характеристики.
Кроме этого сервис помогает подобрать людей, чтобы у них не было противоречий как между собой, так и с задачей.
Основной акцент сделан на визуальном отображении, автоматизации процесса анализа и подсказках.

Создание команды:
* описание
* требуемый DISC
* нужные роли
* мотивация по Герчикова

Просмотр команды:
* список участников с их параметрами
* возможность удалить участника
* сила команды по DISC
* мотивация команды
* роли и их заполненность
* оценить команду по DISK (критические и рискованные проблемы)
* оценить команду по Гречкову
* ознакомиться со тем что нужно, чтобы компенсировать проблему
* составить отчет по команде с помощью LLM (GigaChat)
* посмотреть отчеты
* собрать команду автоматически (под капотом алгоритм). Не берет людей у которых несовместимость по DISC или Гречкову
* посмотреть всех пользователей и их параметры
* возможность добавить участника
